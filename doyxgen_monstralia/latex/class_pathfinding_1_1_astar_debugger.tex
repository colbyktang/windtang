\doxysubsection{Detailed Description}
Debugger for the A$\ast$ \mbox{\hyperlink{namespace_pathfinding}{Pathfinding}} Project. This class can be used to profile different parts of the pathfinding system and the whole game as well to some extent. 

Clarification of the labels shown when enabled. All memory related things profiles {\bfseries{the whole game}} not just the A$\ast$ \mbox{\hyperlink{namespace_pathfinding}{Pathfinding}} System.~\newline

\begin{DoxyItemize}
\item Currently allocated\+: memory the GC (garbage collector) says the application has allocated right now.
\item Peak allocated\+: maximum measured value of the above.
\item Last collect peak\+: the last peak of \textquotesingle{}currently allocated\textquotesingle{}.
\item Allocation rate\+: how much the \textquotesingle{}currently allocated\textquotesingle{} value increases per second. This value is not as reliable as you can think it is often very random probably depending on how the GC thinks this application is using memory.
\item Collection frequency\+: how often the GC is called. Again, the GC might decide it is better with many small collections or with a few large collections. So you cannot really trust this variable much.
\item Last collect fps\+: F\+PS during the last garbage collection, the GC will lower the fps a lot.
\item F\+PS\+: current F\+PS (not updated every frame for readability)
\item Lowest F\+PS (last x)\+: As the label says, the lowest fps of the last x frames.
\item Size\+: Size of the path pool.
\item Total created\+: Number of paths of that type which has been created. Pooled paths are not counted twice. If this value just keeps on growing and growing without an apparent stop, you are are either not pooling any paths or you have missed to pool some path somewhere in your code.
\end{DoxyItemize}
