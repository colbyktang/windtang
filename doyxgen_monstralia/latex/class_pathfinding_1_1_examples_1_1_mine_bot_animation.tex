\doxysubsection{Detailed Description}
Animation helper specifically made for the spider robot in the example scenes. The spider robot (or mine-\/bot) which has been copied from the Unity Example Project can have this script attached to be able to pathfind around with animations working properly.~\newline
This script should be attached to a parent Game\+Object however since the original bot has Z+ as up. This component requires Z+ to be forward and Y+ to be up.~\newline


A movement script (e.\+g \mbox{\hyperlink{class_pathfinding_1_1_a_i_path}{A\+I\+Path}}) must also be attached to the same Game\+Object to actually move the unit.

Animation is handled by this component. The Animation component refered to in \mbox{\hyperlink{class_pathfinding_1_1_examples_1_1_mine_bot_animation_a35703ccd4704ef42c1b628421ebd512a}{anim}} should have animations named \char`\"{}awake\char`\"{} and \char`\"{}forward\char`\"{}. The forward animation will have it\textquotesingle{}s speed modified by the velocity and scaled by \mbox{\hyperlink{class_pathfinding_1_1_examples_1_1_mine_bot_animation_a79b7b28b49e0dc96d2fbf3f96df7ff9e}{animation\+Speed}} to adjust it to look good. The awake animation will only be sampled at the end frame and will not play.~\newline
When the end of path is reached, if the \mbox{\hyperlink{class_pathfinding_1_1_examples_1_1_mine_bot_animation_a4daf91c6dfd884c976a2985edc203a48}{end\+Of\+Path\+Effect}} is not null, it will be instantiated at the current position. However a check will be done so that it won\textquotesingle{}t spawn effects too close to the previous spawn-\/point. \mbox{[}Open online documentation to see images\mbox{]}
