\doxysubsection{Detailed Description}
Helper class for easily updating graphs. 

The \mbox{\hyperlink{class_pathfinding_1_1_graph_update_scene}{Graph\+Update\+Scene}} component is really easy to use. Create a new empty Game\+Object and add the component to it, it can be found in Components-\/-\/$>$Pathfinding-\/-\/$>$\mbox{\hyperlink{class_pathfinding_1_1_graph_update_scene}{Graph\+Update\+Scene}}.~\newline
When you have added the component, you should see something like the image below. \mbox{[}Open online documentation to see images\mbox{]} The region which the component will affect is defined by creating a polygon in the scene. If you make sure you have the Position tool enabled (top-\/left corner of the Unity window) you can shift+click in the scene view to add more points to the polygon. You can remove points using shift+alt+click. By clicking on the points you can bring up a positioning tool. You can also open the \char`\"{}points\char`\"{} array in the inspector to set each point\textquotesingle{}s coordinates manually. \mbox{[}Open online documentation to see images\mbox{]} In the inspector there are a number of variables. The first one is named \char`\"{}\+Convex\char`\"{}, it sets if the convex hull of the points should be calculated or if the polygon should be used as-\/is. Using the convex hull is faster when applying the changes to the graph, but with a non-\/convex polygon you can specify more complicated areas.~\newline
The next two variables, called \char`\"{}\+Apply On Start\char`\"{} and \char`\"{}\+Apply On Scan\char`\"{} determine when to apply the changes. If the object is in the scene from the beginning, both can be left on, it doesn\textquotesingle{}t matter since the graph is also scanned at start. However if you instantiate it later in the game, you can make it apply it\textquotesingle{}s setting directly, or wait until the next scan (if any). If the graph is rescanned, all \mbox{\hyperlink{class_pathfinding_1_1_graph_update_scene}{Graph\+Update\+Scene}} components which have the Apply On Scan variable toggled will apply their settings again to the graph since rescanning clears all previous changes.~\newline
You can also make it apply it\textquotesingle{}s changes using scripting. {\ttfamily  Get\+Component$<$\+Graph\+Update\+Scene$>$().Apply (); } The above code will make it apply its changes to the graph (assuming a \mbox{\hyperlink{class_pathfinding_1_1_graph_update_scene}{Graph\+Update\+Scene}} component is attached to the same Game\+Object).

Next there is \char`\"{}\+Modify Walkability\char`\"{} and \char`\"{}\+Set Walkability\char`\"{} (which appears when \char`\"{}\+Modify Walkability\char`\"{} is toggled). If Modify Walkability is set, then all nodes inside the area will either be set to walkable or unwalkable depending on the value of the \char`\"{}\+Set Walkability\char`\"{} variable.

Penalty can also be applied to the nodes. A higher penalty (aka weight) makes the nodes harder to traverse so it will try to avoid those areas.

The tagging variables can be read more about on this page\+: tags (view in online documentation for working links) \char`\"{}\+Working with tags\char`\"{}.

Note\+: The Y (up) axis of the transform that this component is attached to should be in the same direction as the up direction of the graph. So if you for example have a grid in the XY plane then the transform should have the rotation (-\/90,0,0).

\doxysubsubsection{\texorpdfstring{penaltyDelta}{penaltyDelta}}
{\footnotesize\ttfamily int Pathfinding.\+Graph\+Update\+Scene.\+penalty\+Delta}



Penalty to add to nodes. Usually you need quite large values, at least 1000-\/10000. A higher penalty means that agents will try to avoid those nodes more. 

Be careful when setting negative values since if a node gets a negative penalty it will underflow and instead get really large. In most cases a warning will be logged if that happens.

Update node\textquotesingle{}s walkability and connectivity using physics functions. For grid graphs, this will update the node\textquotesingle{}s position and walkability exactly like when doing a scan of the graph. If enabled for grid graphs, \mbox{\hyperlink{class_pathfinding_1_1_graph_update_scene_ab46e783d9018568430f80231d9a0bc56}{modify\+Walkability}} will be ignored. 

For Point Graphs, this will recalculate all connections which passes through the bounds of the resulting Graph Update Object using raycasts (if enabled).
